# Contributing to ClaimHeart

First off, thank you for considering contributing to ClaimHeart! It's people like you that make this powerful AI claims adjudication platform possible.

## How to Contribute

1. **Fork the repository** and create your branch from `main`.
2. **Setup your environment** using the instructions in `docs/local-setup.md`. This will guide you through starting the Docker orchestration.
3. **Make your changes**. 
    - Ensure your code adheres to standard conventions.
    - Python: Use Ruff and Black (checked via pre-commit).
    - Next.js: Ensure ESLint and Prettier rules pass.
4. **Run the test suite**. Ensure all existing tests pass via `make test`.
5. **Issue a Pull Request**. Describe your changes in detail, link to any relevant issues, and wait for code review.

## Code of Conduct
Please be respectful and considerate of others when submitting code, discussing architectural design, or submitting issues. ClaimHeart is a medical and health-tech system, requiring a high level of rigorous collaboration, safety, and empathy.

## Need Help?
Check out our architecture narratives in the `docs/` folder, or open a discussion thread if you're not sure where to start.
